# Settings Screen Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Material 3 settings screen with toggles for Split tunnel bypass, network requests, privacy mode, port range presets, theme selection, and relocated GitHub/permissions actions.

**Architecture:** New SettingsActivity with hand-crafted XML layout using MaterialSwitch, ChipGroup, and MaterialCardView. MainActivity gets a TopAppBar with settings icon. SharedPreferences stores all settings; checkers read prefs at runtime.

**Tech Stack:** Kotlin, Material 3, SharedPreferences, AppCompatDelegate (theme switching)

**Spec:** `docs/superpowers/specs/2026-04-09-settings-design.md`

```kotlin
fun maskIp(ip: String): String {
    val ipv4Parts = ip.split(".")
    if (ipv4Parts.size == 4 && ipv4Parts.all { it.toIntOrNull() != null }) {
        return "${ipv4Parts[0]}.${ipv4Parts[1]}.*.*"
    }
    val ipv6Parts = ip.split(":")
    if (ipv6Parts.size == 8) {
        return "${ipv6Parts[0]}:${ipv6Parts[1]}:${ipv6Parts[2]}:${ipv6Parts[3]}:*:*:*:*"
    }
    return "*.*.*.*"
}
```
